## Colors

The `Button{:inline}` component color can be defined using the `color{:inline}` prop.

Available colors: `primary{:inline}`, `secondary{:inline}`, `accent{:inline}`, `neutral{:inline}`,
`ghost{:inline}`, and `danger{:inline}`.

## Sizes

The `Button{:inline}` component size can be defined using the `size{:inline}` prop.

Available sizes: `sm{:inline}`, `md{:inline}`, `lg{:inline}`, `icon{:inline}`.

## Icon

The `Button{:inline}` component can be defined as an icon when the `size` prop is set to
`icon{:inline}`.

The icon should be as a child of the `Button{:inline}` component.

<Note title="Accessiblity" type="tip">
  `aria-label{:inline}` is recommended when using icons as buttons. This ensures that screen readers
  announce the purpose of the button.
</Note>

<Showcase>
  <div slot="example" class="flex flex-wrap gap-6 justify-center">
    <Button color="primary" size="icon" aria-label="Heart"><Heart /></Button>
    <Button color="secondary" size="icon" aria-label="Bell"><Bell /></Button>
    <Button color="accent" size="icon" aria-label="Pencil"><Pencil /></Button>
    <Button color="neutral" size="icon" aria-label="Settings"><Settings /></Button>
    <Button color="ghost" size="icon" aria-label="Power"><Power /></Button>
    <Button color="danger" size="icon" aria-label="Trash"><Trash /></Button>
  </div>

```astro
---
import Button from '@/components/ui/button.astro';

---
```
