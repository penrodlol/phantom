## Scrollable

By default, the `Body{:inline}` component will handle horizontal and vertical overflow when the
content exceeds the size of the dialog.

The `Header{:inline}` and `Footer{:inline}` components will remain fixed at the top and bottom of
the dialog, respectively.

<Showcase>
 <Dialog.Root slot="example">
    <Button slot="dialog-trigger">Open Dialog</Button>
    <Dialog.Header>
      <Dialog.Title>Dialog Title</Dialog.Title>
      <Dialog.Description>Dialog Description</Dialog.Description>
    </Dialog.Header>
    <Dialog.Body class="h-screen w-screen">Dialog Body</Dialog.Body>
    <Dialog.Footer>
      <Button color="ghost" size="sm" x-on:click="$root.close()">Cancel</Button>
      <Button size="sm" x-on:click="$root.close()">Save</Button>
    </Dialog.Footer>
  </Dialog.Root>

## Animate

By default, `Dialog{:inline}` component opening is animated with a fade-in and pop-in effect.

If graphics performance is a concern, the animation can be disabled using the `animate{:inline}`
prop.

<Note type="tip">
  A common use case is for dialogs which contain large amounts of content. Disabling animations can
  prevent the dialog from appearing to stutter when opening.
</Note>

<Showcase>
 <Dialog.Root slot="example" animate={false}>

    <Dialog.Header>
      <Dialog.Title>Dialog Title</Dialog.Title>

## Backdrop

By default, `Dialog{:inline}` component has a backdrop which closes the dialog when clicked.

This can be disabled using the `backdropClose{:inline}` prop.
