## Elevation

The `TabList{:inline}` component elevation can be defined using the `elevation{:inline}` prop.

Available elevations: `flat{:inline}`, `raised{:inline}`.

## Position

The `TabList{:inline}` component position can be defined using the `position{:inline}` prop.

Available positions: `start{:inline}`, `end{:inline}`, and `center{:inline}`.

<Note type="warning">
  The `position` prop will have no effect if the `grow` prop is set to `true`.
</Note>

## Activation

The `TabPanel{:inline}` component, by default, will be activated automatically when the
corresponding `Tab` is focused.

This can be changed to manual activation by setting the `activation{:inline}` prop to
`manual{:inline}` on the `Root{:inline}` component. By setting this prop, the `TabPanel{:inline}`
will only be activated when the `Tab{:inline}` is focused and the user
presses<Command value="Enter" /> or <Command value="Space" />.

Available activations: `automatic{:inline}`, `manual{:inline}`.

<Showcase>
  <Tabs.Root slot="example" activation="manual">
    <Tabs.TabList>
      <Tabs.Tab value="tab-1">Tab 1</Tabs.Tab>
      <Tabs.Tab value="tab-2">Tab 2</Tabs.Tab>
      <Tabs.Tab value="tab-3">Tab 3</Tabs.Tab>
    </Tabs.TabList>
    <Tabs.TabPanel value="tab-1">Tab 1 Content</Tabs.TabPanel>
    <Tabs.TabPanel value="tab-2">Tab 2 Content</Tabs.TabPanel>
    <Tabs.TabPanel value="tab-3">Tab 3 Content</Tabs.TabPanel>
  </Tabs.Root>

## Default Value

The `Root{:inline}` component can define the default active `Tab{:inline}` using the
`defaultValue{:inline}` prop.

<Note>The defaultValue must match the value prop of one of the Tab components.</Note>
